  <action>
Create `src/app/admin/inventory/page.tsx` as a client component (`'use client'`). Follow the exact structural patterns from `src/app/admin/products/page.tsx`.

**Component structure:**

1. **Imports:** Use `useEffect`, `useState`, `useCallback`, `useRef` from React. Import `useRouter` from `next/navigation`. Import `authClient` from `@/lib/auth-client`. Import shadcn `Input`, `Button` from their established paths. Import `DataTable`, `StatusBadge`, `DataTablePagination` from `@/components/ui/data-table`. Import `PageContainer`, `PageHeader` from `@/components/dashboard/*`. Import `Search`, `BarChart3` from `lucide-react`.

2. **Types:** Define `InventoryProduct` interface: `{ id, name, sku, stock, lowStockThreshold, category: { id, name } | null, price: string }`. Define `Pagination` interface (same shape as products page).

3. **Stock status function:** Define `getStockStatus(product)` returning `{ variant: "active" | "low-stock" | "out-of-stock", label: string }`:
   - `stock === 0` -> `{ variant: "out-of-stock", label: "Out of Stock" }`
   - `stock >= 1 && stock <= 10` -> `{ variant: "low-stock", label: "Low Stock" }`
   - `stock > 10` -> `{ variant: "active", label: "Healthy" }`

4. **State:**
   - `products: InventoryProduct[]` (default [])
   - `pagination: Pagination` (default page=1, pageSize=20)
   - `searchQuery: string` (default "")
   - `lowStockFilter: boolean` (default false)
   - `loading: boolean` (default true)
   - `debounceTimer: useRef`

5. **fetchInventory function:** Build URL with query params from current state. Base URL: `/api/inventory`. Append `page`, `pageSize`, `search` (if non-empty), `lowStock=true` (if filter active). Fetch and set `products` + `pagination`. Wrap in try/catch with console.error. Set loading true/false.

6. **Auth guard useEffect:** On mount, call `authClient.getSession()`. Redirect to `/login` if no session. Redirect to `/pos` if role !== "manager". Then call `fetchInventory(1)`.

7. **handleSearchChange:** Same debounce pattern as products page — 300ms delay. When search is empty, call fetchInventory(1) immediately.

8. **handleLowStockToggle:** Toggle `lowStockFilter` state, reset to page 1.

9. **useEffect on lowStockFilter:** When `lowStockFilter` changes, call `fetchInventory(1)` (depends on lowStockFilter + fetchInventory).

10. **Columns definition (useMemo):**
    - Product name + SKU (same cell pattern as products page — initial letter in rounded box, name, SKU in mono font)
    - Category (muted text, "—" fallback for null)
    - Stock (right-aligned, tabular-nums, colored: red for 0, amber for 1-10)
    - Status (StatusBadge using getStockStatus — the badge component already handles "active", "low-stock", "out-of-stock" variants)

11. **JSX return:**
    ```
    <PageContainer>
      <PageHeader title="Inventory" description="Monitor stock levels across all products" />

      {/* Toolbar */}
      <div className="flex flex-col gap-3 sm:flex-row sm:items-center">
        {/* Search input — same pattern as products page */}
        <div className="relative flex-1">
          <Search icon>
          <Input placeholder="Search products..." ... />
        </div>

        {/* Low Stock toggle — pill button */}
        <Button
          variant={lowStockFilter ? "default" : "outline"}
          size="sm"
          onClick={handleLowStockToggle}
        >
          Show Low Stock Only
        </Button>
      </div>

      {/* DataTable */}
      <DataTable columns={columns} data={products} loading={loading} skeletonRows={8} />

      {/* Pagination */}
      <div className="rounded-xl border border-border/60 bg-card shadow-sm">
        <DataTablePagination ... />
      </div>
    </PageContainer>
    ```

12. **Low-stock filter button styling:** When active (`lowStockFilter === true`), use `variant="default"` (primary bg). When inactive, use `variant="outline"`. This matches the UI-SPEC accent usage rule.

13. **Empty state handling:** The DataTable component already handles empty state with a centered "No results found" message. For the specific copy from the UI-SPEC ("No products match your filter."), use the DataTable's built-in empty state since it provides the same UX. The DataTable's empty state is sufficient.

14. **No actions column:** Unlike the products page, the inventory page has NO edit/deactivate actions (read-only monitoring per UI-SPEC). The columns should be: Product (name+SKU), Category, Stock, Status. No actions dropdown.

**Design tokens to use (from UI-SPEC):**
- Page background: `bg-background`
- Card/table: `bg-card` with `shadow-sm` and `border-border/60`
- Input: `bg-background` with `pl-9` for search icon
- Stock text color: `text-red-600` for stock=0, `text-amber-600` for stock 1-10, default for stock > 10
- Status badges: Use existing StatusBadge with variants "active" (green), "low-stock" (amber), "out-of-stock" (red)
- Table header: uppercase meta text via DataTable's built-in styling
- Row hover: DataTable handles via `hover:bg-accent/50`
