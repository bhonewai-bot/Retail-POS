  <action>
Create `src/app/api/inventory/route.ts` with a single `GET` export handler. Follow the exact patterns from `src/app/api/products/route.ts` (import style, error handling with try/catch, console.error with emoji prefix, Decimal serialization).

**Query parameters:**
- `page` (default 1, min 1) — page number
- `pageSize` (default 20, min 1, max 100) — items per page
- `lowStock` (optional, `"true"` string) — when present, filter to products where `stock <= 10` (fixed threshold per UI-SPEC)
- `search` (optional string) — case-insensitive contains match on `name` OR `sku`

**Prisma query structure:**
Build a dynamic `where` clause object. Start with `isActive: true`. If `lowStock=true`, add `stock: { lte: 10 }`. If `search` is non-empty, add `OR: [{ name: { contains: search, mode: 'insensitive' } }, { sku: { contains: search, mode: 'insensitive' } }]`.

Use `Promise.all` to fetch `[products, total]` in parallel (same pattern as products route). Include `{ category: true }` in the query. Order by `stock: 'asc'` (lowest first — matches UI-SPEC default sort). Apply `skip`/`take` for pagination.

**Serialization:** Map each product to spread all fields, serialize `price` and `cost` to strings via `.toString()` (matching products route pattern). Include `category` nested object.

**Response shape:**
```json
{
  "products": [...],
  "pagination": { "page": 1, "pageSize": 20, "totalPages": 5, "total": 100 }
}
```

**Error handling:** Wrap in try/catch, log `"❌ GET /api/inventory error:"` + error, return `{ error: 'Failed to fetch inventory' }` with status 500.
