**File 1: `src/app/api/inventory/adjustments/route.ts`**

**POST /api/inventory/adjustments — Create an adjustment**

Accept JSON body: `{ productId: number, type: string, quantity: number, reason?: string }`

Validation:
- `productId` must be a positive integer — return 400 `{ error: "Product ID is required" }` if missing/invalid
- `type` must be one of: `stock-receipt`, `damage`, `count-adjustment`, `return`, `other` — return 400 `{ error: "Invalid adjustment type" }` if not in list
- `quantity` must be a non-zero integer — return 400 `{ error: "Quantity must be a non-zero integer" }` if missing, zero, or not integer
- If `reason` is provided, it must be a string (trim it)

Implementation:
1. Verify the product exists with `prisma.product.findUnique({ where: { id: productId } })`. Return 404 `{ error: "Product not found" }` if null.
2. Calculate new stock: `product.stock + quantity`. If result < 0, return 400 `{ error: "Insufficient stock" }`.
3. Use `prisma.$transaction` to atomically: (a) create the InventoryAdjustment record, (b) update the Product stock. The transaction ensures both succeed or both fail.
4. For `userId`: Since this is a demo POS without full auth integration on API routes, use a hardcoded fallback. Query for the first manager user: `prisma.user.findFirst({ where: { role: "manager" } })`. Use their `id` as `userId`. This is acceptable for a demo — the auth middleware protects the admin UI, and the API is called from that protected UI.
5. Return 201 with the created adjustment object (spread fields, serialize any Decimal fields).

Response shape:
```json
{
  "id": 1,
  "type": "stock-receipt",
  "quantity": 50,
  "reason": "Monthly restock",
  "productId": 5,
  "userId: "user_abc123",
  "createdAt": "2026-07-07T..."
}
```

**GET /api/inventory/adjustments — List adjustment history**

Query parameters:
- `page` (default 1, min 1)
- `pageSize` (default 20, min 1, max 100)
- `productId` (optional number) — filter by product
- `type` (optional string) — filter by adjustment type

Prisma query:
- `findMany` with `include: { product: { select: { id: true, name: true, sku: true } }, user: { select: { id: true, name: true } } }`
- Dynamic where clause: start with empty `{}`, add `productId` filter if provided, add `type` filter if provided
- Order by `createdAt: 'desc'` (newest first)
- Apply `skip`/`take` for pagination
- Use `Promise.all` for `[adjustments, total]` pattern

Response shape:
```json
{
  "adjustments": [...],
  "pagination": { "page": 1, "pageSize": 20, "totalPages": 5, "total": 100 }
}
```

Return 404 if not found. Otherwise return the adjustment object.
