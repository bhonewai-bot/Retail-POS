**File 1: `src/components/inventory/adjustment-dialog.tsx`**

This is a modal dialog for creating inventory adjustments. Use the `dialog` shadcn component (install with `npx shadcn add dialog` if not already installed).

**Props interface:**
```typescript
interface AdjustmentDialogProps {
  open: boolean;
  onOpenChange: (open: boolean) => void;
  product: { id: number; name: string; sku: string; stock: number } | null;
  onAdjustmentCreated: () => void; // callback to refresh parent data
}
```

**Form state:**
- `type: string` â€” dropdown select, options: `stock-receipt`, `damage`, `count-adjustment`, `return`, `other`
- `quantity: string` â€” text input (parsed as integer on submit). Positive = add stock, negative = remove stock
- `reason: string` â€” optional textarea

**Form layout:**
- Dialog title: "Adjust Stock â€” {product.name}"
- Product info row: Show product name, SKU, current stock (read-only)
- Type selector: Use a `<select>` or shadcn `Select` component with the 5 type options. Display labels: "Stock Receipt", "Damage", "Count Adjustment", "Return", "Other"
- Quantity input: Number input. Show helper text: "Positive to add stock, negative to remove"
- Reason textarea: Optional, placeholder "Enter reason for adjustment..."
- Buttons: "Cancel" (outline variant) and "Submit Adjustment" (default variant)

**Submit handler:**
1. Parse quantity as integer. Validate it is non-zero. Show inline error if invalid.
2. POST to `/api/inventory/adjustments` with `{ productId, type, quantity: parsedInt, reason: reason.trim() || undefined }`
3. On success: call `onAdjustmentCreated()` callback, close dialog, reset form
4. On error: display error message from response in the dialog

**Styling:** Follow the shadcn Dialog patterns. Use `bg-background` for dialog content, `text-foreground` for text, standard spacing tokens.
