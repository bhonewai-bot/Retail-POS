  <name>Task 1: Create seed script for demo auth users</name>
  <files>scripts/seed-auth.ts</files>
  <action>
Create `scripts/seed-auth.ts` — a TypeScript script that seeds the database with two demo users.

Implementation per RESEARCH.md Pattern 4 and D-03:

1. Imports:
   - `import { PrismaClient } from "../app/generated/prisma/client"` (matches existing scripts pattern from scripts/test-database.ts)
   - `import { PrismaPg } from "@prisma/adapter-pg"`
   - `import { hashPassword } from "better-auth/crypto"`
   - `import "dotenv/config"`

2. Create Prisma client instance (same pattern as lib/prisma.ts but inline for script):
   ```
   const adapter = new PrismaPg({ connectionString: process.env.DATABASE_URL! });
   const prisma = new PrismaClient({ adapter });
   ```

3. Main `seed()` async function:

   a. Hash both passwords using Better Auth's `hashPassword` (per D-04: 8+ chars, and RESEARCH "Don't Hand-Roll" — use library hashing):
      - `const managerHash = await hashPassword("demo1234")`
      - `const cashierHash = await hashPassword("demo1234")`

   b. Create manager user (per D-03: admin@demo.com, role: manager):
      Use `prisma.user.upsert` with `where: { email: "admin@demo.com" }` for idempotency:
      - `create`: name "Admin User", email "admin@demo.com", role "manager", isActive true
      - `update`: (empty object — skip if already exists)
      - `include`: { accounts: true }

   c. Create manager account (password stored in account table, NOT user table — per RESEARCH Pitfall 2):
      Only create if the user has no existing "credential" account. Use conditional logic:
      - Check `manager.accounts.some(a => a.providerId === "credential")`
      - If no credential account exists, create one via `prisma.account.create`:
        - id: `crypto.randomUUID()`
        - accountId: `manager.id.toString()`
        - providerId: "credential"
        - userId: `manager.id`
        - password: `managerHash`

   d. Create cashier user (per D-03: cashier@demo.com, role: cashier):
      Same upsert pattern:
      - `create`: name "Cashier User", email "cashier@demo.com", role "cashier", isActive true
      - `update`: (empty)
      - `include`: { accounts: true }

   e. Create cashier account (same conditional pattern as manager).

   f. Console output per project conventions (emoji + status messages):
      - `console.log("Seeding auth users...")`
      - `console.log("Created/updated: admin@demo.com (role: manager)")`
      - `console.log("Created/updated: cashier@demo.com (role: cashier)")`
      - `console.log("Both passwords: demo1234")`

4. Error handling:
   - Wrap in try/catch/finally
   - Catch: `console.error("Seed failed:", error); process.exit(1)`
   - Finally: `await prisma.$disconnect()`

5. Call `seed()` at the bottom of the file.

6. Add a `package.json` script entry — NOT in this task, but note for the executor: after creating the file, add to package.json scripts: `"seed:auth": "tsx scripts/seed-auth.ts"`. Use Edit to add this line to the scripts section of package.json.

Important:
- Use `upsert` not `create` so the script is idempotent (can be run multiple times)
- Better Auth stores passwords in the `account` table, not the `user` table
- The `accountId` field should be the user's ID as a string (Better Auth convention)
- The `providerId` must be "credential" for email+password authentication
- Use `crypto.randomUUID()` for account IDs (available in Node.js 19+ without import, or use `import { randomUUID } from "crypto"`)
