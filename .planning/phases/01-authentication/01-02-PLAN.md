<task type="auto">
  <name>Task 1: Extend Prisma schema with Better Auth models</name>
  <files>prisma/schema.prisma</files>
  <action>
Edit `prisma/schema.prisma` to add Better Auth required models and update the User model. Do NOT remove any existing models (Category, Product, Customer, Order, OrderItem).

**1. Add the session model** (after the existing OrderItem model, before User):

```
model session {
  id        String   @id
  expiresAt DateTime
  token     String   @unique
  ipAddress String?
  userAgent String?
  userId    Int

  user User @relation(fields: [userId], references: [id], onDelete: Cascade)

  @@index([userId])
}
```

**2. Add the account model** (after session):

```
model account {
  id                   String    @id
  accountId            String
  providerId           String
  userId               Int
  password             String?
  accessToken          String?
  refreshToken         String?
  idToken              String?
  accessTokenExpiresAt DateTime?
  refreshTokenExpiresAt DateTime?
  scope                String?

  user User @relation(fields: [userId], references: [id], onDelete: Cascade)

  @@unique([accountId, providerId])
  @@index([userId])
}
```

**3. Add the verification model** (after account):

```
model verification {
  id         String   @id
  identifier String
  value      String
  expiresAt  DateTime

  @@unique([identifier, value])
}
```

**4. Update the existing User model:**

Add these fields/relations to the existing `model User` block:
- Add `sessions session[]` relation
- Add `accounts account[]` relation
- Keep all existing fields (id, name, email, role, isActive, createdAt, updatedAt) unchanged
- Keep the existing `@@index([email])`

The final User model should look like:

```
model User {
  id        Int    @id @default(autoincrement())
  name      String
  email     String @unique
  role      String @default("cashier")
  isActive  Boolean @default(true)

  sessions session[]
  accounts account[]

  createdAt DateTime @default(now())
  updatedAt DateTime @updatedAt

  @@index([email])
}
```

Important notes:
- The `session` and `account` model names are lowercase — this matches Better Auth's Prisma adapter expectation (the skill says "config uses ORM model name, not DB table name")
- `userId` in session/account is `Int` to match User's `id` type (autoincrement Int)
- Cascade deletes ensure cleanup when a user is removed

<task type="auto">
  <name>Task 2: Apply schema migration and regenerate Prisma client</name>
  <files>app/generated/prisma/</files>
  <action>
Run the following commands in order to apply the schema changes to the database and regenerate the Prisma client:

1. Push the schema to the database:
   Run: `npx prisma db push`
   This applies schema changes without creating a migration file (suitable for development).
   Wait for completion — it should print "Your database is now in sync with your Prisma schema."

2. Regenerate the Prisma client to include the new models:
   Run: `npx prisma generate`
   This updates the generated client in `app/generated/prisma/` with types for session, account, and verification.

3. Verify the database has the new tables by running:
   Run: `npx prisma db execute --stdin <<< "SELECT table_name FROM information_schema.tables WHERE table_schema = 'public' AND table_name IN ('session', 'account', 'verification', 'User');"`
   Expected output should list all four tables.

   If `prisma db execute` is not available, verify by checking the generate output or run the existing database test script:
   Run: `npm run db:test`
   The test script should still pass (it tests existing models which are unchanged).

If any step fails:
- `prisma db push` failure: Check DATABASE_URL in .env is correct and PostgreSQL is reachable
- `prisma generate` failure: Re-read the schema for syntax errors
- Table creation failure: Check that the database user has CREATE TABLE permissions
