1. Install better-auth as a dependency (not devDependency):

2. Generate a 32-character base64 secret for Better Auth:

   Capture the output.

3. Read .env file. Add or update these two environment variables:

   - `BETTER_AUTH_URL` = `http://localhost:3000`

4. Verify: Run `npm ls better-auth` to confirm the package is installed and listed in package.json dependencies.

<task type="auto">
  <name>Task 2: Create auth server and client configuration files</name>
  <files>lib/auth.ts, lib/auth-client.ts</files>
  <action>
Create two files per the Better Auth skill guide (better-auth-best-practices).

**File 1: lib/auth.ts** — Server-side auth configuration.

Import `betterAuth` from `better-auth`, import `prismaAdapter` from `better-auth/adapters/prisma`, and import the existing prisma singleton from `./prisma`.

Configure with these exact settings (per D-01, D-02, D-04):
- `database`: use `prismaAdapter(prisma, { provider: "postgresql" })`
- `emailAndPassword.enabled`: `true`
- `session.expiresIn`: `60 * 60 * 24 * 365 * 100` (100 years — effectively no expiration, per D-02)
- `session.updateAge`: `60 * 60 * 24` (refresh every 24 hours)
- `session.cookieCache.enabled`: `true`
- `session.cookieCache.maxAge`: `60 * 5` (5 minute cache)
- `session.cookieCache.strategy`: `"compact"` (smallest payload for POS performance)
- `user.additionalFields`: add `role` field with `type: "string"`, `required: true`, `defaultValue: "cashier"`

Export the `auth` object as a named export.

Do NOT set `secret` or `baseURL` in config — Better Auth reads them from `BETTER_AUTH_SECRET` and `BETTER_AUTH_URL` env vars automatically (per security best practices skill).

**File 2: lib/auth-client.ts** — Client-side auth utilities.

Import `createAuthClient` from `better-auth/react`.

Call `createAuthClient()` with no arguments (uses default base URL). Export the result as `authClient` named export.

Both files use TypeScript with strict mode. Use `import` syntax (not require). Follow project conventions: camelCase for variables, named exports for multi-export modules.
  </action>

  </verify>
  <done>lib/auth.ts exports a configured Better Auth instance with Prisma adapter, email+password, no-expiration sessions, and cookie cache. lib/auth-client.ts exports a React auth client.</done>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Client to API | Auth endpoints accept credentials — must validate and rate-limit |
| Server to Database | Auth queries for session/user — Prisma provides parameterized queries |
| Cookie to Server | Session cookies sent with requests — Better Auth handles CSRF and signing |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Tampering | Auth secret | mitigate | Generate high-entropy secret via openssl, store in .env (gitignored), never commit |
| T-01-02 | Information Disclosure | Session cookies | mitigate | Better Auth uses httpOnly+secure cookies by default; cookieCache strategy "compact" adds HMAC |
| T-01-03 | Elevation of Privilege | Role field | mitigate | role defaults to "cashier" in config; managers must be explicitly assigned via seed |
| T-01-SC | Tampering | npm/better-auth install | mitigate | Package legitimacy verified in RESEARCH.md — better-auth is [OK] with 150k+/week downloads |
</threat_model>
