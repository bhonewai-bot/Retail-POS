<task type="auto">
  <name>Task 1: Create Next.js middleware for route protection</name>
  <files>src/middleware.ts</files>
  <action>
Create `src/middleware.ts` at the project root of the src directory.

This middleware enforces AUTH-02 (role-based access) and AUTH-04 (cashiers blocked from admin routes). Per the RESEARCH.md Pattern 3, it decodes the session from the cookie to check role without hitting the database (cookie cache strategy "compact" from Plan 01).

Implementation:

1. Import `NextRequest`, `NextResponse` from `next/server`.

2. Define the `middleware` async function that receives `request: NextRequest`.

3. Extract the session cookie value: `const session = request.cookies.get("better-auth.session_token")?.value;`
   Note: Better Auth's default cookie name is `better-auth.session_token`. If the cookieCache compact strategy is used, the session data is base64url-encoded in this cookie.

4. Define route arrays:
   - `protectedPaths`: `["/pos", "/admin"]` â€” routes requiring authentication
   - `adminPaths`: `["/admin"]` â€” routes requiring manager role

5. Check if the current pathname starts with any protected path.

6. Logic flow:
   a. **Not authenticated + protected route:** If no session cookie and path is protected, redirect to `/login` using `NextResponse.redirect(new URL("/login", request.url))`.
   b. **Authenticated + login page:** If session cookie exists and pathname is exactly `/login`, redirect to `/pos` using `NextResponse.redirect(new URL("/pos", request.url))`.
   c. **Admin route + session:** If path is admin-only and session exists:
      - Try to decode the session cookie. Better Auth compact format is `base64url.data.hmac`. Split the cookie by ".", take the first part (base64url-encoded JSON), decode it with `Buffer.from(parts[0], "base64url")`, parse as JSON.
      - Extract `sessionData.user?.role` from the decoded object.
      - If role is NOT "manager", redirect to `/pos` (cashier cannot access admin, per D-05).
      - If role IS "manager", continue (per D-06).
      - On any decode/parse failure, redirect to `/login` (invalid session).
   d. **Default:** Return `NextResponse.next()` â€” allow the request through.

7. Export the `config` object with a `matcher` that covers the protected and login routes:
   ```typescript
   export const config = {
     matcher: ["/pos/:path*", "/admin/:path*", "/login"],
   };
   ```

Important: The middleware runs on the Edge runtime by default in Next.js 16. Use only Web APIs (no Node.js `fs`, `crypto`, etc.). `Buffer` and `TextDecoder` are available in Edge. `JSON.parse` is available. Do NOT import any server-side libraries or Prisma.

For the cookie decoding, handle these edge cases:
- Missing cookie -> redirect to login
- Cookie with fewer than 3 parts (not in compact format) -> redirect to login
- JSON parse failure -> redirect to login
- Missing user or role in decoded data -> redirect to login
